## Features Overview

### 1. **Overview Dashboard** (Home Page)

- Real-time testing statistics
- Primary objectives checklist
- Quality gates tracking
- Test environment setup status
- Project information

### 2. **Functional Testing**

Organized by modules:

- **Authentication & Roles** - Login, RBAC tests
- **Customer Search** - BVN, phone, name searches
- **Duplicate Prevention** - Same-day duplicate checks
- **API Integration** - Credit Registry & First Central

### 3. **Non-Functional Testing**

Four categories:

- **Performance** - Load time, concurrency, optimization
- **Security** - OAuth, HTTPS, encryption, injection prevention
- **Usability** - UI/UX, error messages, accessibility
- **Compatibility** - Cross-browser and mobile testing

### 4. **Defect Tracking**

- Add/Edit/Delete defects
- Severity levels (Critical, High, Medium, Low)
- Status tracking (Open, In Progress, Resolved, Closed)
- Assignment to team members
- Steps to reproduce
- Real-time statistics

### 5. **Success Metrics**

- Track 4 key metrics against targets
- Inline editing of actual results
- Visual progress indicators
- Achievement rate calculation
- Status tracking (Met, Not Met, Pending)

### 6. **Reports & Export**

Multiple export formats:

- **Summary Report** (.txt) - Overview and statistics
- **Test Cases** (.csv) - All test cases spreadsheet
- **Defects** (.csv) - Defect log spreadsheet
- **All Data** (.json) - Complete data export
- **Print** - Browser print functionality

## Usage Guide

### Adding Test Results

1. Navigate to Functional or Non-Functional Testing
2. Click on the desired module tab
3. Click the "Edit" button on any test case
4. Fill in:
   - Actual Result
   - Status (Pass/Fail/Pending/Blocked)
   - Comments
5. Click "Save"

### Logging Defects

1. Go to "Defect Tracking" page
2. Click "Add Defect" button
3. Fill in the form:
   - Bug ID (required)
   - Severity level
   - Module (required)
   - Description (required)
   - Steps to reproduce
   - Status
   - Assigned To
4. Click "Add Defect"

### Tracking Metrics

1. Navigate to "Success Metrics" page
2. Click on "Actual Result" field to edit
3. Enter the measured result
4. Select status from dropdown (Met/Not Met/Pending)
5. Data saves automatically

### Exporting Reports

1. Go to "Reports & Export" page
2. Choose export format:
   - Click "Export" on desired format
   - File downloads automatically
3. View summary preview on the page
4. Use "Print" button for hard copies

## Troubleshooting

### Common Issues

#### 1. Module Not Found Errors

```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```

#### 2. shadcn Component Import Errors

```bash
# Reinstall specific component
npx shadcn-ui@latest add button
```

#### 3. TypeScript Errors

```bash
# Restart TypeScript server in VS Code
Cmd/Ctrl + Shift + P â†’ "TypeScript: Restart TS Server"
```

#### 4. Tailwind Classes Not Working

- Verify `tailwind.config.ts` includes correct content paths
- Check `globals.css` has Tailwind directives
- Restart dev server

#### 5. LocalStorage Not Persisting

- Check browser privacy settings
- Ensure localStorage is enabled
- Clear browser cache if corrupted

### Build Errors

If build fails:

```bash
# Clean build cache
rm -rf .next
npm run build
```

### Port Already in Use

```bash
# Kill process on port 3000
# Mac/Linux:
lsof -ti:3000 | xargs kill

# Windows:
netstat -ano | findstr :3000
taskkill /PID <PID> /F
```
